Simon Sapin
 

When I joined Kozea two years ago, they told me something along these lines: “We need PDF. We want to use CSS for this. WebKit’s page breaks are not great. Let’s build our own browser engine!”

My first reaction was “No, that crazy. We’re not gonna do that.” Still, I’ve been working on WeasyPrint since then. It’s an open-source implementation of CSS that renders web pages to PDF files. It is, of course, written all in Python.

Working on WeasyPrint lead me to spend a lot of time with CSS specifications and on the www-style@w3.org mailing-list. Nowadays I’m a member of W3C’s CSS Working Group and editor of the CSS Paged Media module.

In addition to WeasyPrint, I have a bunch of cool open-source projects that you should check out, as does Kozea. I also sometimes blog technical stuff.

Just before PyCon Taiwan, I will leave Kozea and join Mozilla to work on Firefox, Servo, and CSS specs. But I will certainly not forget WeasyPrint!

  • WeasyPrint − bring the web to PDF and paper
